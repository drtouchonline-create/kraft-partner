# CLAUDE.md — Правила работы с проектом Kraft GmbH Proposal

## Superpowers Skills

- Новая секция / идея / компонент → `brainstorming` сначала
- Баг → `systematic-debugging`
- Большая задача → `writing-plans` → `executing-plans`
- Перед «готово» / перед показом клиенту → `verification-before-completion`

---

## Главное правило

Перед добавлением новой секции, карточки или блока — рассказать, что планируется: структура, текст, CTA. Только после подтверждения — писать код.

---

## Технические правила

- Проект — один самодостаточный HTML-файл: `kraft-partner-proposal.html`
- Без внешних библиотек и CDN
- Язык пропозала: **немецкий**
- Общение с пользователем: **русский**

---

## Дизайн

- Цветовая схема: **navy/gold** (тёмно-синий фон, золотые акценты)
- Стиль: премиальный, корпоративный

---

- Клиент: **Kraft GmbH**, Мюнхен — специалист по лизинговому ПО в Европе
- Цель: показать Kraft возможности AI-разработки (вайб-кодинг) → больше клиентов, лучше перед банками
- Уникальное преимущество Kraft: единственный в Германии специалист по всему стеку лизингового ПО
- Презентер: **Touch Digital** (hallo@touchdigital.de)
